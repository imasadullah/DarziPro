# Project Rules

You are building a production-grade Tailor Shop Management System.

Always follow:

- Angular 20 standalone components.
- Strict TypeScript mode.
- Reusable components only.
- Feature-based folder structure.
- No duplicate code.
- Use Angular Material.
- Use TypeORM.
- Use SQLite.
- Use Signals where appropriate.
- Mobile responsiveness is not required.
- Desktop-first design.
- Generate tests for all business logic.
- Use clean architecture principles.
- Use repository pattern.
- Use lazy loaded feature modules.
- Never use mock data unless requested.
- Never create placeholder screens.
- Use Angular Signals for state management.
- Do not use NgRx unless explicitly requested.
